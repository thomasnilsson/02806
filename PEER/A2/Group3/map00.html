Part 2: Visualizing Murders over Time

This visualization is divided in three main parts, which are realted between each other. However, they are intended to be understood as one single dashboard; reason why there is only one title in the top left (to avoid redundancy and excesssive text). The top left of the dashboard shows a map of New York City divided by boroughs with teal circles at the locations where there has been a murder (from 2006 to 2016, both included). The top right is a histogram of the number of murders that occurred in NYC by hour of the day. For example, the first column shows the number of murders between midnight and 1am. Finally, the bottom graphic is a line chart with the number of murders in NYC from 2006 to 2016.

We have implemented two interactions for this visualization. First of all, a brush in the timeline that allows for the selection of a range of dates. Both the circles in the map and the histogram are filtered with the date range to show only the corresponding data. Second, we have implemented and "Animate" button that sets the brush to a width equal to the number of days set in the text input box next to the button and then transitions the brush from beginning to end. To re-set the brush, you can double-click anywhere in the timeline line chart. The transition when re-setting is a bit less smooth than the rest due to the big amount of data processed.
